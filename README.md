# 3D Raycasting Game Engine (Cub3D)

이 프로젝트는 레이캐스팅 기술을 사용하여 2D 맵을 3D 게임처럼 보이게 만드는 간단한 게임 엔진입니다. 전설적인 게임인 울펜슈타인 3D(Wolfenstein 3D)에서 영감을 받았습니다.

## 주요 기능

- **레이캐스팅**: 2D 맵 데이터를 기반으로 3D 시점을 렌더링합니다.
- **맵 파싱**: `.cub` 확장자를 가진 파일에 정의된 맵을 읽고 해석합니다.
- **플레이어 조작**: 키보드 입력에 따라 플레이어가 맵 안에서 움직이고 시점을 회전할 수 있습니다.
- **텍스처**: 벽에 텍스처를 매핑하여 사실적인 그래픽을 구현합니다.
- **스프라이트 및 추가 기능 (보너스)**: 문, 적, 마우스 시점 조작 등 다양한 보너스 기능이 포함될 수 있습니다.

## 실행 방법

### 의존성

이 프로젝트는 `MiniLibX` 라이브러리를 사용합니다. `minilibx-linux` 디렉토리에 포함되어 있거나, 시스템에 맞게 설치해야 합니다.

### 컴파일

프로젝트 루트 디렉토리에서 `make` 명령어를 실행하여 프로젝트를 컴파일합니다.

```bash
make
```

보너스 기능을 포함하여 컴파일하려면 다음 명령어를 사용하세요.

```bash
make bonus
```

### 실행

컴파일이 완료되면 실행 파일(`cub3D`)이 생성됩니다. 게임을 실행하려면 맵 파일(`.cub`)을 인자로 전달해야 합니다.

```bash
./cub3D maps/good/subject_map.cub
```

보너스 버전 실행:

```bash
./cub3D_bonus maps/good/subject_map.cub
```

## 프로젝트 구조

```
.
├── include/          # 헤더 파일
├── src/              # 소스 코드
│   ├── parsing/      # 맵 파싱 관련 코드
│   ├── rendering/    # 렌더링 및 레이캐스팅 코드
│   ├── utils/        # 유틸리티 함수
│   └── bonus/        # 보너스 기능 관련 코드
├── libft/            # 42 공용 라이브러리
├── minilibx-linux/   # MiniLibX 그래픽 라이브러리
├── maps/             # 테스트용 맵 파일 (.cub)
│   ├── good/         # 유효한 맵
│   └── bad/          # 유효하지 않은 맵 (오류 테스트용)
├── textures/         # 게임에 사용될 텍스처 및 스프라이트 이미지
├── Makefile          # 컴파일 설정 파일
└── main.c            # 메인 함수
```

## 맵 파일 형식 (`.cub`)

맵 파일은 다음 요소들로 구성됩니다.

- **방향별 텍스처 경로**: `NO`, `SO`, `WE`, `EA` 키워드와 함께 텍스처 파일 경로를 지정합니다.
- **바닥 및 천장 색상**: `F` (바닥), `C` (천장) 키워드와 함께 RGB 색상 값을 지정합니다.
- **맵 구조**: `0`(빈 공간), `1`(벽), `N`/`S`/`W`/`E`(플레이어 시작 위치 및 방향)으로 구성된 2D 지도를 작성합니다.
